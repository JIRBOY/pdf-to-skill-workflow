# TwinCAT 3 ADS 编程技能

基于 TE1000 手册 (版本 2.4.0, 2025-06-04) 提取的 ADS 编程指南。

## 何时触发

- 编写与 TwinCAT PLC/NC/CNC 进行 ADS 通信的代码
- 使用 ADS .NET API、ADS DLL 或原始 AMS/TCP 协议
- 通过 ADS 读写 PLC 变量、I/O 过程映像、NC 轴参数
- 订阅设备通知 (Device Notifications)
- 使用 Sum 命令批量读写
- 排查 ADS 通信错误（返回码、路由、防火墙）

## 1. ADS 基础概念

### 1.1 客户端-服务器架构

- TwinCAT 系统中每个软件模块视为独立的"虚拟设备"
- 消息由 **消息路由器 (Message Router)** 通过 ADS 接口交换
- 通信模式：
  - **已确认服务**：请求(Request) -> 指示(Indication) -> 响应(Response) -> 确认(Confirmation)
  - **通知服务**：客户端登记后，服务器主动推送数据（低开销）
  - **异步服务**：客户端请求后继续运行，通过回调获取结果
  - **同步服务**：客户端调用线程阻塞直到结果返回

### 1.2 设备寻址

每个 ADS 设备由两个标识符唯一标识：

| 标识符 | 说明 |
|--------|------|
| **AMS NetId** | 6字节，格式如 `172.16.17.10.1.1`，前4字节可自选，后2字节为子网掩码 |
| **AMS Port** | 2字节，固定或动态分配 |

### 关键 AMS 端口速查

| 端口 | 设备 |
|------|------|
| 1 | ADS 路由器 |
| 30 | 授权服务器 |
| 100 | 记录器 |
| 120 | EtherCAT 设备 |
| 801 | TC2 PLC 运行时系统 1 |
| 851 | TC3 PLC 运行时系统 1 |
| 852 | TC3 PLC 运行时系统 2 |
| 500 | NC 轴（端口 500 + 轴ID） |
| 501 | NC 安全功能/内部端口（SAF port 501） |
| 10000 | 系统服务 |
| 11000 | NC 控制系统 |
| 20000-25999 | ADS 服务器保留 |
| 26000-26999 | 客户私有端口范围 |
| 32768-65535 | ADS 客户端保留 |

> **重要**：PLC 运行时系统端口为 851-854（TC3），纯客户端首次访问时自动获取动态端口（32768+）。

### 1.3 防火墙配置

| 方向 | 本地端口 | 协议 | 用途 |
|------|----------|------|------|
| 传入 | 48898 | TCP | ADS 通信 |
| 传入 | 48899 | UDP | 广播搜索 |
| 传入 | 8016 | TCP | 安全 ADS |

## 2. AMS/TCP 包结构

```
┌─────────────────────────────────┐
│ AMS/TCP 报头 (6 字节)            │
│  ├── 保留 (2 字节) = 0           │
│  └── 长度 (4 字节)               │
├─────────────────────────────────┤
│ AMS 报头 (32 字节)               │
│  ├── AMSNetId 目标 (6 字节)       │
│  ├── AMSPort 目标 (2 字节)        │
│  ├── AMSNetId 来源 (6 字节)       │
│  ├── AMSPort 来源 (2 字节)        │
│  ├── 命令 ID (2 字节)             │
│  ├── 状态旗标 (2 字节)            │
│  ├── 数据长度 (4 字节)            │
│  ├── 错误代码 (4 字节)            │
│  └── 调用 ID (4 字节)             │
├─────────────────────────────────┤
│ ADS 数据 (n 字节)                │
└─────────────────────────────────┘
```

### 2.1 命令 ID

| 命令 ID (Hex) | 命令 | 说明 |
|---------------|------|------|
| 0x0001 | ADS ReadDeviceInfo | 读取设备名称和版本 |
| 0x0002 | ADS Read | 从设备读取数据 |
| 0x0003 | ADS Write | 向设备写入数据 |
| 0x0004 | ADS ReadState | 读取设备状态 |
| 0x0005 | ADS WriteControl | 更改设备状态 |
| 0x0006 | ADS AddDeviceNotification | 创建设备通知 |
| 0x0007 | ADS DeleteDeviceNotification | 删除设备通知 |
| 0x0008 | ADS DeviceNotification | 服务器主动推送数据 |
| 0x0009 | ADS ReadWrite | 同时读写 |

### 2.2 状态旗标

- Bit 0: 0=请求, 1=响应
- Bit 2: 必须为 1（ADS 命令）
- Bit 6: 传输协议 (0=TCP, 1=UDP)
- 其他位必须为 0

请求旗标值: `0x0004`
响应旗标值: `0x0005` (0x0004 | 0x0001)

## 3. ADS 命令详细规范

### 3.1 ADS Read (0x0002)

**请求结构:**
```
| 索引组 (4B) | 索引偏移 (4B) | 读取长度 (4B) |
```

**响应结构:**
```
| 结果码 (4B) | 返回长度 (4B) | 数据 (nB) |
```

### 3.2 ADS Write (0x0003)

**请求结构:**
```
| 索引组 (4B) | 索引偏移 (4B) | 写入长度 (4B) | 数据 (nB) |
```

**响应结构:**
```
| 结果码 (4B) |
```

### 3.3 ADS ReadWrite (0x0009)

**请求结构:**
```
| 索引组 (4B) | 索引偏移 (4B) | 读长度 (4B) | 写长度 (4B) | 写入数据 (nB) |
```

### 3.4 ADS AddDeviceNotification (0x0006)

**请求结构:**
```
| 索引组 (4B) | 索引偏移 (4B) | 长度 (4B) | 传输模式 (4B) |
| 最大延迟 (4B, ms) | 循环时间 (4B, ms) | 保留 (16B, 全0) |
```

**响应结构:**
```
| 结果码 (4B) | 通知句柄 (4B) |
```

> **限制**：每个设备建议不超过 550 个通知。超出时使用 Sum 命令或增大有效负载。

### 3.5 ADS ReadDeviceInfo (0x0001)

**响应结构:**
```
| 结果码 (4B) | 主版本 (1B) | 次版本 (1B) | 构建号 (2B) | 设备名称 (16B) |
```

## 4. PLC ADS 索引组

PLC 索引组分为四个全局范围：

| 索引组范围 | 说明 |
|-----------|------|
| 0x00000000-0x00000FFF | 保留 |
| **0x00001000** | PLC ADS 参数范围 |
| **0x00002000** | PLC ADS 状态范围 |
| **0x00003000** | PLC ADS 单元功能范围 |
| **0x00004000** | PLC ADS 服务（含 %M 字段访问） |
| 0x00006000-0x0000EFFF | PLC ADS 扩展保留 |
| **0x0000F000** | 通用 TwinCAT ADS 系统服务（含 I/O 过程映像） |

### 4.1 PLC 内存范围访问 (0x4000 组)

| 索引组 | 索引偏移 | 访问 | 说明 |
|--------|----------|------|------|
| 0x4020 | 0x0000-0xFFFF | R/W | %M 字段，字节偏移 |
| 0x4021 | 0x0000-0xFFFFFFFF | R/W | %MX 字段，位地址 = 字节号*8 + 位号 |
| 0x4030 | 0x0000-0xFFFFFFFF | R/W | 保留数据范围 (%R) |
| 0x4040 | 0x0000-0xFFFFFFFF | R/W | 数据范围 (%D) |
| 0x4035 | 0x00000000 | R | 保留范围字节长度 |
| 0x4045 | 0x00000000 | R | 数据范围字节长度 |

### 4.2 I/O 过程映像 (0xF000 组)

| 索引组 | 索引偏移 | 访问 | 说明 |
|--------|----------|------|------|
| 0xF020 | 0x1F400+ | R/W | %I 物理输入，字节偏移 |
| 0xF021 | 0xFA000+ | R/W | %IX 物理输入位，偏移 = 0xFA000 + 字节号*8 + 位号 |
| 0xF025 | 0x00000000 | R | 输入过程映像字节长度 |
| 0xF030 | 0x3E800+ | R/W | %Q 物理输出，字节偏移 |
| 0xF031 | 0x1F4000+ | R/W | %QX 物理输出位，偏移 = 0x1F4000 + 字节号*8 + 位号 |
| 0xF035 | 0x00000000 | R | 输出过程映像字节长度 |

### 4.3 符号访问 (0xF000 组)

| 索引组 | 索引偏移 | 说明 |
|--------|----------|------|
| 0xF003 | 0x00000000 | `GET_SYMHANDLE_BYNAME` — 为变量名分配句柄 |
| 0xF005 | 句柄值 | `READ_SYMVAL_BYHANDLE` / `WRITE_SYMVAL_BYHANDLE` |
| 0xF006 | 0x00000000 | `RELEASE_SYMHANDLE` — 释放句柄 |

> **符号访问流程**：先 0xF003 获取句柄 -> 用 0xF005 读写 -> 最后 0xF006 释放句柄。

## 5. Sum 命令 (批量操作)

Sum 命令允许在一次 ADS 请求中执行多个子操作，最多 500 个子命令。

### 5.1 Sum Read (0xF084)

**写入数据:**
```
[IG1, IO1, Len1, IG2, IO2, Len2, ..., IGn, IOn, Lenn]
```

**读取数据:**
```
[Result1, Result2, ..., Resultn, Data1, Data2, ..., Datan]
```

### 5.2 Sum Write (0xF081)

**写入数据:**
```
[IG1, IO1, WLen1, ..., IGn, IOn, WLenn, WData1, ..., WDataN]
```

**读取数据:**
```
[Result1, Result2, ..., Resultn]
```

### 5.3 Sum ReadWrite (0xF082)

**写入数据:**
```
[IG1, IO1, RLen1, WLen1, ..., IGn, IOn, RLenn, WLenN, WData1, ..., WDataN]
```

**读取数据:**
```
[Result1, RLen1, ..., Resultn, RLenn, RData1, ..., RDataN]
```

### 5.4 Sum AddDeviceNotification (0xF085)

一次请求注册多个通知，返回多个通知句柄。

### 5.5 Sum DeleteDeviceNotification (0xF086)

一次请求删除多个通知句柄。

## 8. 编程最佳实践

### 8.1 符号访问优于直接索引

```
// 推荐：使用符号名（可读性强，不易出错）
// 1. 获取句柄: 0xF003, 写入变量名
// 2. 读写值:   0xF005 + 句柄
// 3. 释放句柄: 0xF006
```

直接索引 (0x4020, 0x4040 等) 仅适用于已知固定偏移的场景。

### 8.2 批量操作减少开销

- 多个变量读写优先使用 **Sum 命令**（最多 500 个子命令）
- 多个通知注册优先使用 `0xF085` Sum AddDeviceNotification
- 周期性轮询应改用 **通知机制**（设备主动推送，开销更低）

### 8.3 通知使用约束

- 每个设备最多 550 个通知
- 传输模式选择：
  - 循环模式：按 cycleTime 检查值变化
  - 按需模式：值变化时立即通知
- 必须在不使用时用 `ADS DeleteDeviceNotification` (0x0007) 清理句柄

### 8.4 死区时间补偿

对于高精度位置控制，编码器（0x5000 组）和驱动器（0x7000 组）支持死区时间补偿：
- 模式 0: 关闭
- 模式 1: 开启（带速度）
- 模式 2: 开启（带速度和加速度）

### 8.5 NC 安全注意事项

- 索引偏移 `0x0000002D`（CST 扭矩模式切换）涉及轴意外运动风险
- 启用轴前必须确认运行模式为位置相关模式 (CSV/CSP)
- 轴功能写入操作多数仅适用于 SAF Port 501
- 重新初始化编码器时**不能启用控制器**（防止位置跳变）
